网络与通信

OSI七层网络模型

层 描述 数据单位 常见协议 设备
应用层 也称为应用实体，一般指的是应用程序，主要负责确定通信对象，并确保有足够的资源用于通信 FTP, HTTP
表示层 一般负责数据的编码以及转化，负责进行数据的压缩、解压、加密、解密等
会话层 主要负责在网络中的两个节点之间建立、维护、控制会话，区分不同的会话，以及提供单工、半双工、全双工三种通信模式服务 NFS(Network File System,网络文件系统)RPC(Remote Procedure Call,远程过程调用)
传输层 主要负责分割，组合数据，实现端到端的链接 段 TCP, UDP
网络层 主要负责管理网络地址，定位设备，决定路由 数据包 IP, RIP, OSPF 路由器
数据链路层 主要负责物理传输的准备，包括物理地址寻址，CRC校验 帧 交换机
物理层 规定了激活，维持，关闭通信端点之间的机械特性、电气特性、功能特性以及过程特性 集线器，双绞线，同轴电缆

路由

从一个端口接收到数据包，根据数据包的目的地址进行定向并转发到另外一个端口。

直连路由

去往路由器的接口地址所在网段的路径，不需要再配置路由。

静态路由

由管理员手动配置的的固定路由，优点是简单、高效，缺点是适应性差，一般适用于网络规模不大，拓扑结构固定的网络中。

动态路由

路由器之间相互通信，传递路由信息，利用收集到的路由信息更新路由表，自动生成路由表。能实时地使用网络结构的变化，适应性强，适用于网络规模大，网络拓扑复杂的网络。

telnet

Internet远程登录服务，为用户提供了在本地计算机上完成远程主机工作的能力，一般使用方法为通过终端登录到远程主机。

TCP与UDP的比较

UDP(用户数据报协议)

(1)UDP是无连接的：在发送数据前不需要建立连接，发送数据结束时也没有连接可释放

(2)UDP尽最大努力交付：不保证可靠交付

(3)UDP是面向报文的

(4)UDP没有拥塞控制：网络出现拥塞不会是源主机的发送速率降低。

(5)UDP支持一对一，一对多，多对一，多对多的交互通信。

(6)UDP的首部开销小：只有8字节，TCP首部为20字节。

提供无连接的通信，不能保证数据包被发送到目标地址。

UDP报头的四个域：源端口号，目的端口号，数据报长度，校验和

TCP(控制传输协议)

(1)面向连接：应用程序在使用TCP协议之前，必须先建立TCP连接，在传送数据完毕后，必须释放已经建立的TCP连接。

(2)每一条TCP连接只能是点对点的，一对一的

(3)TCP提供可靠交付。

(4)TCP提供全双工通信。

(5)面向字节流

TCP可靠传输的实现

  1. 滑动窗口

  2. 超时重传

  3. 选择确认SACK

TCP的流量控制

  1. 滑动窗口

TCP的拥塞控制

  1. 慢开始和拥塞避免

  2. 快重传和快恢复

流量控制：让发送方的发送速率不要太快，要让接收方来得及接收。

拥塞控制：防止过多的数据注入到网络中，这样可以使网络中的路由器或链路不致过载。

流量控制是一个端到端的问题，是接收端抑制发送端发送数据的速率，以便使接收端来得及接收。拥塞控制是一个全局性的过程，涉及到所有的主机、所有的路由器，以及与降低网络传输性能有关的所有因素。
