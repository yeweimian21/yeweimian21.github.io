2.网络配置：将em1和em2网卡端口bond为bond0，使用一个IP

(1)#vim /etc/network/interfaces
#
#
#
auto lo #环回网址
iface lo inet loopback

up ifenslave bond0 em1 em2
down ifenslave bond0 -d em1 em2
bond-mode 0 #模式0
bond-mmimon 100

auto bond0
iface bond0 inet static
address 10.10.4.7
netmask 255.255.255.0
gateway 10.10.4.1
dns-nameserver 202.106.0.20
(2)#vim /etc/modules
#
#
#
bonding
(3)#modprobe bonding,然后重启服务器。
(4)创建和删除 Bond(完成1-3步直接重启就能自动创建bond0)：

添加新的 Bond（bond0）：

echo +bond0 > /sys/class/net/bonding_masters

删除现有的 Bond（bond0）：

echo -bond0 > /sys/class/net/bonding_masters

显示所有现有的 Bond：

cat /sys/class/net/bonding_masters

(5)添加和删除 Slave：

使用文件 /sys/class/net/bond/bonding/slaves，可以使接口从属于某个 Bond。 该文件的语义与 bonding_masters 文件相同。

使接口 em1,em2 从属于 bond0：

echo +em1 > /sys/class/net/bond0/bonding/slaves

echo +em2 > /sys/class/net/bond0/bonding/slaves

从 Bond（bond0）中释放 Slave（em1，em2）：

echo -em1 > /sys/class/net/bond0/bonding/slaves

#echo -em2 > /sys/class/net/bond0/bonding/slaves

启用bond0

ifconfig bond0 up

(6)网卡端口加入BOND以后端口不需要设置IP地址,在BOND上配置IP地址等(interfaces中已经对bond0配置完毕)
(7)将em1，em2添加进bond0的slaves之后，就能暂时上网了（重启之后就又不能了），趁现在安装ifenslave

#apt-get update

#apt-get install ifenslave

以后重启的话，网络也可以一直生效了。

(8)确认bond0状态
cat /proc/net/bonding/bond0

提示up即成功：
MII Status: up

而down表示未启用可能没有auto（自动启用），可以尝试手工启动：
ifconfig bond0 up

(9)键盘不能用试试重启虚拟控制台。
