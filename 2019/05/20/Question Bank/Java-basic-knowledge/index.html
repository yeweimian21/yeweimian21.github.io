10. 继承、封装、多态

(1)继承：子类可以继承父类的方法和实例变量，并且子类可以修改或者增加新的方法。

(2)封装：将客观事物抽象成类，每个类对自身的数据和方法实行保护。类可以把自己的数据和方法只让可信的类或者对象操作，对某些信息进行隐藏。

(3)多态：允许不同类的对象对同一消息做出响应。多态性语言具有灵活、抽象、行为共享和代码共享的优势，很好地解决了应用程序方法同名的问题。

11. 内部类

(1)静态内部类

可以不依赖于外部类实例而被实例化，不能与外部类有相同的名字，只能访问外部类的静态属性和静态方法。

(2)成员内部类

可以访问外部类的静态和非静态的属性和方法，但是不能定义静态的属性和方法。只有在外部的类被实例化后，这个内部类才能被实例化。

(3)局部内部类

定义在一个代码块内的类，作用范围为其所在的代码块。同局部变量一样，不能被public, protected, private, static修饰。

  • 局部静态内部类：在外部类的静态方法或者静态初始化代码块中。基本特性与静态内部类相同。
  • 局部内部类：在外部类的非静态方法或者非静态初始化代码块中。基本特性与成员内部类相同。

(4)匿名内部类

匿名内部类没有类名，不使用关键字class, extends, implements，没有构造方法，必须继承其他类或者实现其他接口。一般应用于GUI编程中实现事件处理。

13. final关键字

使用范围 描述
final 类 类不可以被继承
final 属性 属性值不可改变
final 方法 方法不可被覆盖
final 参数 参数在这个方法内不允许被修改
  1. 一个类不能既被声明为abstract，又被声明为final
  2. 有一种inline(内联)机制，当调用一个被声明为final的方法时，直接将方法主体插入发哦调用出，而不是进行方法调用，这样能提高程序的效率。(类似于C++语言中的内联inline)

14. 构造方法

(1)构造方法必须与类的名字相同，并且不能有返回值

(2)每个类可以有多个构造方法，当没有编写构造方法时，编译器在把源代码编译成字节码的过程中会提供一个默认的无参的空的构造方法，如果编写了构造方法，则编译器不会创建默认的构造方法。

(3)构造方法可以有0个, 1个, 多个参数。

(4)构造方法总是和new操作一起调用，由系统调用，不能由开发者调用。构造方法在对象实例化时会被自动调用，且只运行一次，而普通方法是在程序执行到它时才被调用，可以被该对象调用多次。

(5)构造方法的主要作用是完成对象的初始化工作。

(6)构造方法不能被继承和覆盖，但是能够被重载。

(7)子类可以通过super关键字显式地调用父类的构造方法。

  • 当父类提供了无参的构造方法，子类可以不显式地调用父类的构造方法，默认调用父类的无参构造方法。

  • 当父类没有提供无参的构造方法时，子类的构造方法中必须显式地调用父类的构造方法。

(8)默认的构造方法的修饰符只与当前类的修饰符有关。

(9)在其他构造方法中可以使用this()来调用类的默认构造方法。

17. 抽象类和接口

(1) 抽象类

  1. 抽象类可以有私有变量和私有方法
  2. 在抽象类的子类中，实现方法必须是相同或者更高的访问级别(public, protected, private)
  3. 抽象类的子类为父类的所有抽象方法提供具体实现，否则子类仍然是抽象类
  4. 抽象类的抽象方法不能用private, static, synchronized, native等修饰。

(2) 接口

  1. 接口是公开的，不能有有私有变量和私有方法

相同点：

  1. 都不能被实例化
  2. 接口的实现类只有实现了类接口的全部方法才能被实例化，抽象类的子类只有实现了抽象类的全部抽象方法才能被实例化。

不同点：

  1. 抽象类可以有方法的定义和实现，接口只能有方法的定义
  2. 一个类只能继承一个抽象类，可以实现多个接口

19.多态

(1)编译时多态：重载

(2)运行时多态：覆盖

20.覆盖

子类覆盖父类的函数，

  1. 子类中的覆盖方法必须和父类中被覆盖的方法有相同的函数名，参数和返回值类型
  2. 子类中的覆盖方法所抛出的异常必须和父类中被覆盖的方法所抛出的异常相同或者是其子类
  3. 不能用子类的静态方法覆盖父类的实例方法，不能用子类的实例方法覆盖父类的静态方法

21.重载

  1. 重载是通过不同的参数列表来区分的
  2. 不能通过方法的访问权限、返回值类型和抛出的异常类型来进行重载

22.重载和覆盖的区别

  1. 覆盖是子类和父类之间的关系，重载是同一个类中的方法之间的关系
  2. 覆盖是一对方法之间的关系，重载是多个方法之间的关系
  3. 覆盖要求参数列表相同，重载要求参数列表不同
  4. 覆盖是对象的类型来选择调用的方法体，重载是根据调用时的实参列表和形参列表来选择调用的方法体。

23.finally关键字

  1. finally块里的代码会在return前执行
  2. 如果try-finally或者catch-finally中都有return，则finally块中的return语句将会覆盖别处的return语句，最终只调用finally语句块中的return。
  3. 当有异常发生后，catch和finally块进行处理后程序就结束了，finally块后面即使有代码也不会执行。如果没有异常发生，在执行完finally块的代码后，后面的代码还会继续执行。

24.static关键字

作用：

  1. 为类分配单一的存储空间，与创建对象的个数无关。

  2. 使某个属性或者方法与类而不是对象关联起来，在不创建对象的情况下可以直接通过类来调用属性或者方法。

(1)static成员变量

  1. 静态变量属于类，在内存中只有一个副本，所有实例都指向同一个内存地址。只要静态变量所在的类被加载，这个静态变量就会被分配空间。

  2. Java中没有全局的概念，但是可以通过static关键字来达到全局的效果。

  3. Java中不能在方法体中定义static变量(这与C++不同)

(2)static成员方法

(3)static代码块

  1. JVM在加载类的时候会执行static代码块，如果有多个static代码块，JVM将会按顺序来执行。
  2. static代码块经常被用来初始化静态变量
  3. static代码块只会被执行一次

(4)static内部类

  1. 可以不依赖于外部类实例而被实例化
  2. 不能与外部类有相同的名字
  3. 只能访问外部类的静态属性和静态方法。

异常

分类

Error和Exception拥有共同的父类：Throwable

Error

Error表示在运行期间出现了非常严重的错误,并且错误是不可回复的，属于JVM层次的严重错误，会导致程序终止运行。

例如：OutOfMemoryError, ThreadDeath

Exception

表示可以恢复的异常，编译器可以捕捉到。

检查异常(Checked Exception)

发生在编译阶段，Java编译器强制程序去捕获这种类型的异常，把可能出现这些异常的代码放到try块中，把对异常处理的代码放到catch中。

例如： IO异常，SQL异常

运行时异常(Runtime Exception)

编译器没有强制对其进行捕获，如果不对这几种异常进行处理，当出现这种异常时，会由JVM来处理。

例如：NullPointException, IndexOutOfBoundException, ClassCastException, ArithmeticException(算数异常), ArrayStoreException(数据存储异常), BufferOverflowException(缓冲区溢出异常)

流

序列化(Serializable)

在分布式环境下，当进行远程通信时，无论是何种类型的数据，都会以二进制序列的形式在网络上进行传送。序列化是一种将对象转换成字节序列的过程，用于解决在对对象流进行读写操作时所引发的问题。序列化可以将对象的状态写在流里进行网络传输，或者保存到文件、数据库等系统里，并在需要的时候把该流读取出来重新构造成一个相同的对象。

  1. 如果一个类能序列化，那么它的子类也能够序列化

  2. static、transient类型的属性不能被序列化

  3. 子类实现了Serializable接口，父类没有，父类的属性不能被序列化。

  3. LinkedList是采用双向链表来实现的，插入删除的时候不需要移动数据。LinkedList不是线程安全的。

守护线程与用户线程

守护线程(Daemon Thread)

守护线程又被称为“服务线程”，“精灵线程”，“后台线程”，是指当程序运行时，在后台提供一种通用服务的线程。守护线程一般具有较低的优先级，但是并非只由JVM内部提供，用户也可以自己设置守护线程，方法是在thread.start()之前调用thread.setDaemon(true)，false表示用户线程。当一个守护线程中产生了其他线程，那么这些新产生的线程默认还是守护线程。典型的守护线程有GC(垃圾回收器)。

用户线程和守护线程几乎一样，区别在于如果用户线程全部退出运行，只剩下守护线程存在，程序就会终止，杀死所有守护线程，JVM退出。只要有任何非守护线程还在运行，程序就会不会终止。

线程

  1. 线程的同步：线程按照预定的先后次序运行

  2. 线程的调度：按照特定机制为多个线程分配CPU的使用权。

  3. 线程的就绪：这个线程已经有了除CPU外所有的资源，在等待获取CPU，一旦获取到CPU的控制权就可以立即执行。

  4. 线程的互斥：某一资源同时只允许一个访问者对其进行访问，但是互斥无法限制访问者对资源的访问顺序。

死锁

是指两个或者两个以上的进程在执行过程中，因争夺资源而造成的一种互相等待的现象，若无外力作用，它们都将无法推进下去。

JDBC

标准SQL规范的4个事务隔离级别

  1. Read Uncommitted(读未提交)
  2. Read committed(读提交)
  3. Repeated Read(可重复读)
  4. Serializable(序列化)

JDBC的5中事务隔离级别

|事务|名称|出现
TRANSACTION_NONE 不支持事物
TRANSACTION_READ_UNCOMMITTED 未提交读 脏读，不可重复读，虚读
TRANSACTION_READ_COMMITTED 已提交读 不可重复读，虚读
TRANSACTION_REPEATABLE_READ 可重复读 虚读
TRANSACTION_SERIALIZABLE 可序列化

读

  1. 脏读：一个事务读取了另一个事务尚未提交的数据。例如：当事务A与事务B并发执行时，当事务A更新后，事务B查询读取到事务A尚未提交的数据，此时事务A回滚，则事务B读取到的数据是无效的脏数据。

  2. 不可重复读：一个事务的操作导致另一个事务前后两次读取到不同的数据。例如：当事务A与事务B并发执行时，当事务B查询读取数据后，事务A更新操作更改事务B查询到的数据，此时事务B再次读取该数据，发现前后两次的数据不一样。

  3. “虚读”：一个事务的操作导致另一个事务前后两次查询的记录数量不同。例如：当事务A与事务B并发执行时，当事务B查询读取数据后，事务A新增或删除了一条满足事务B的查询条件的记录，此时事务B再次查询，发现查询到前次不存在的记录，或者前次的某个记录不存在了。

进程与线程的关系

关系：

  1. 一个线程只能属于一个进程，而一个进程可以有多个线程，但是至少有一个线程。

  2. 资源分配给进程，同一进程上的所有线程共享该进程的资源。

  3. 处理机(CPU,主存，输入输出接口)分配给线程，真正在处理机上运行的是线程。

  4. 线程在执行过程中，需要协调同步，不同进程的线程要利用消息通信的办法实现同步。

相同点：

  1. 都具有ID，一组寄存器，状态

不同点：

  1. 线程是进程的一部分

  2. 线程开销较小

  3. 进程是系统所有资源分配的一个基本单位

进程之间通信的主要方法有：

  1. 信号

  2. 信号量

  3. 管道

  4. 消息

  5. 共享内存

内存管理

  1. 虚拟地址

由程序产生的由段选择符和段内偏移地址两部分组成的地址。

  1. 逻辑地址

用户程序经过编译之后的每个目标模块都以0为基地址顺序编址，在程序中使用的地址都是逻辑地址。

  1. 线性地址

虚拟地址到物理地址变换之间的中间层，是处理器可寻址的内存空间(称为线性地址空间)中的地址。

  1. 物理地址

CPU外部地址总线上的寻址物理内存的地址信号，是地址变换的最终结果，是实际数据存放的地址。

编译器的工作阶段

  1. 词法分析：识别单词

  2. 语法分析：将单词序列组合成各类语法短语，得到语言结构并以树的形式表示

  3. 语义分析：检查语法正确的句子语义是否正确

  4. 中间代码生成(可选)：生成一种既接近目标语言，又与具体语言无关的表示，便于优化与代码生成。

  5. 中间代码优化(可选)：

  6. 目标代码生成：把中间代码变换成特定机器上的绝对指令代码或可重定位的指令代码或汇编指令代码。

  7. 符号表管理：合理组织符号，便于各阶段查找、填写等。

  8. 出错管理：错误的种类，即词法错误、语法错误、静态语义错误、动态语义错误等。

计算模型

  1. 有限状态自动机(Finite State Automation, FSA)：为研究有限内存的计算过程和某些语言类而抽象出的一种计算模型。有限状态自动机拥有有限数量的状态，每个状态可以迁移到零个或多个状态，输入字串决定执行哪个状态的迁移。有限状态自动机可以表示为一个有向图。有限状态自动机是自动机理论的研究对象。

  2. 下推自动机(Push Down Automation, PDA)：是自动机理论中定义的一种抽象模型。下推自动机比有限状态自动机复杂：除了有限状态组成部分外，还包括一个长度不受限制的栈；下推自动机的状态迁移不但要参考有限状态部分，也要参照栈当前的状态；状态迁移不但包括有限状态的变迁，还包括一个栈的出栈或入栈的过程。术语“下推”来自原型机械自动机物理上接触穿孔卡片来阅读其内容的下推动作。术语“确定下推自动机”(Deterministic Push Down Automation, DPDA)指的是识别确定上下文无关语言的抽象计算设备。

  3. 图灵机：又称图灵计算、图灵计算机，是由图灵提出的一种抽象计算模型，它有一条无限长的纸带，纸带分成了一个一个的小方格，每个方格有不同的颜色。有一个机器头在纸带上移来移去，机器头有一组内部状态，还有一些固定的程序。在每个时刻，机器头都要从当前纸带上读入一个方格信息，然后结合自己的内部状态查找程序表，根据程序输出信息到纸带方格上，并转换自己的内部状态，然后进行移动。
