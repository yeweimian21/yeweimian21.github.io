树

基本术语

(1)结点的度：结点所拥有的子树的个数

(2)树的度：树中各结点度的最大值

(3)兄弟：同一双亲的孩子

(4)堂兄弟：双亲在同一层上且不同的结点

(5)结点的层次：从根节点到树中某结点所经路径上的分支数称为该结点的层次，根结点的层次规定为1，其余结点的层次等于其双亲结点的层次加1
(6)树的深度：树中结点的最大层次数

(7)无序树：树中任意一个结点的孩子之间的次序都无关紧要，通常树是指无序树

(8)有序树：树中任意一个节点的孩子结点都按严格的次序排列。二叉树是有序树，因为二叉树中每个孩子结点都确切定义为是左孩子结点还是右孩子结点

(9)森林：m棵树的集合(m>=0)

二叉树

二叉树的定义

(1)非空二叉树：有序树，左子树，右子树

(2)满二叉树：所有分支结点都存在左、右子树，并且所有叶子结点都在同一层上，这样的二叉树称作满二叉树。

(3)完全二叉树：如果具有n个结点的二叉树的结构与满二叉树的前n个结点的结构相同，这样的树称作完全二叉树。

二叉树的存储结构

(1)顺序存储结构

用元素在数组中的下标为该结点在树中的序号。

(2)链式存储结构

每个结点有一个数据域，两个指针域分别指向左孩子和右孩子。

遍历二叉树

(1)先序遍历：根结点->左子树->右子树

(2)中序遍历：左子树->根结点->右子树

(3)后序遍历：左子树->右子树->根结点

索引顺序表的查找(分块查找)

索引顺序查找又称分块查找，是顺序查找的一种改进方法。

基本思想：先将表分为几块，块內无序，块间有序，查找时先确定待查元素所在的块，然后再在块内查找，每个分块要建立一个索引项，其中包含关键字项（该块内的最大关键字）和指针项（指示该块的第一个元素在表中的位置）

分块有序：索引表按关键字有序，即第二块中的所有元素均大于第一块中的所有元素，第三块中的所有元素均大于第二块中的所有元素，以此类推。

算法流程：
step1 先选取各块中的最大关键字构成一个索引表；
step2 查找分两个部分：先对索引表进行二分查找或顺序查找，以确定待查记录在哪一块中；然后，在已确定的块中进行顺序查找。

二叉排序树（二叉查找树）

定义：二叉排序树或者是一棵空树，或者是具有下列性质的二叉树：

  1. 若它的左子树不为空，则左子树的所有节点的值均小于它的根结点的值。
  2. 若它的右子树不为空，则右子树的所有节点的值均大于它的根结点的值。
  3. 它的左右子树分别为二叉排序树。

总结

  1. B树：多路搜索树，每个结点存储M/2到M个关键字，非叶子结点存储指向关键字范围的子结点；所有关键字在整颗树中出现，且只出现一次，非叶子结点可以命中；
  1. B+树：在B-树基础上，为叶子结点增加链表指针，所有关键字都在叶子结点中出现，非叶子结点作为叶子结点的索引；B+树总是到叶子结点才命中；

哈希表

在数据元素的存储位置和其关键字之间建立一个确定的对应关系h，把具有关键字key的数据元素存储在h(key)处，那么在查找时，只要在集合中存在关键字和key相等的数据元素，则一定在h(key)的存储位置上。

哈希表的构造方法

1.直接定址法

哈希函数直接定义为关键字的线性函数值

h(key) = a*key + b

特点：不会产生冲突，但是仅适合关键字集合较小的情况。

2.除数留余法

用关键字key除以哈希表的长度m所得到的余数作为散列地址的方法

h(key) = key % m

3.数字分析法

在给定的关键字中，选取数字分布比较均匀的若干数字位作为哈希地址

特点：适用于关键字值已知的情况。

4.平方取中法

取关键字平方后的中间几位作为哈希地址。

5.折叠法

将关键字分割成位数相同的几部分（最后一部分的位数可以不同），然后取这几部分的叠加和（舍去进位）作为哈希地址。

6. 随机数法

选择一个随机函数，取关键字的随机函数值作为哈希地址

h(key) = random(key)

特点：通常关键字长度不等时采用此法。

排序

插入排序

直接插入排序

基本思想：顺序地把待排序的数据元素按其关键字值的大小插入到已排序数据元素子集合的适当位置。

希尔排序

基本思想：先将整个待排序元素分割成若干个子序列分别进行排序，待整个序列中的元素“基本有序”时，再对全体元素进行一次直接插入排序。

基本方法：设置一个初始增量，以后增量逐渐减少，分组减少，每组元素数量增多，直到最后增量为1，所有记录为一组，再进行一次整体直接插入排序。

交换排序

冒泡排序

基本思路：从数组第1个元素开始，比较相邻的两个元素，如果后一个元素大于前一个元素，则交换两个元素，然后比较第2个和第3个元素，反复这个过程，每趟排序都是将前一趟结果的最后一个元素排除出去。
(排除后面的元素)

快速排序

基本思路：在待排序的n个元素中任取一个作为“基准”，将其余元素分为两组，左边一组的元素均小于等于基准，右边一组的元素均大于等于基准，基准排在两组之间，这个过程称为一趟快速排序。对分成的两组分别重复上述方法，直到所有元素都排到了适当的位置上。

选择排序

简单选择排序

基本思路：首先找出从A[0]到A[len-1]的最小元素A[small]，将A[small]与A[0]元素交换位置，然后找出从A[1]到A[len-1]的最小元素A[small]，将A[small]与A[1]元素交换位置，然后再从A[2]到A[len-1]中查找，以此类推，重复此步骤，在剩下的范围内找出最小值，并将其和数组下标最小的数据交换，逐次缩小要排序的范围。(排除前面的元素)
